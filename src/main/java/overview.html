DebugTrace class has some logging methods.

The following are a sample code used DebugTrace methods and a log of when the sample was executed.
Log bodys are automatically indent.
Sample code:
package sample;

import jp.masatokokubo.debug.DebugTrace;

public class Sample2 {
  // main
  public static void main(String[] args) {
    DebugTrace.enter();
    DebugTrace.print("args", args);

    sub(args);

    DebugTrace.leave();
  }

  // sub
  private static void sub(String[] args) {
    DebugTrace.enter();

    DebugTrace.print("args", args);

    DebugTrace.leave();
  }
}

DebugTrace read DebugTrace.properties file in the classpath on startup. You can specify following properties in the DebugTrace.properties file.

Properties of DebugTrace.properties file:
NameValueDefault
logger Logger
Select from follows.
    Jdk : When you use JDK Logger
    Log4j : When you use Log4J 1
    Log4j2 : When you use Log4J 2
    SLF4J : When you use SLF4J
    System.out : When you use System.out
    System.err : When you use System.err
 System.out
logLevel Log Level
Select from follows.
    default : The lowest level for each logger
    finest, finer, fine, config, info, warning, severe : JDK Logger
    trace, debug, info, warn, error, fatal : Log4J 1 or 2
    trace, debug, info, warn, error : SLF4J
 default (finest: JDK Logger, trace: Log4J 1, 2 or SLF4J)
enterString Log string used after enter method
    parameters:
    %1: The class name of the caller
    %2: The method name of the caller
    %3: The line numnber of the call point
Enter %1$s.%2$s(%3$d)
leaveString Log string used before leave method
    parameters:
    %1: The class name of the caller
    %2: The method name of the caller
    %3: The line numnber of the call point
Leave %1$s.%2$s(%3$d)
threadBoundaryString Threads boundary string
    parameter:
    %1: The thread name
______________________________ %1$s ______________________________
classBoundaryString Classes boundary string
    parameter:
    %1: The class name
____ %1$s ____
indentString A indent string
    "\\s" is change to a space character
\\s\\s
limitString String to represent that it has exceeded the limit ...

varNameValueSeparator Separator between the variable name and value \\s=\\s
keyValueSeparator Separator between the key and value for Map object :\\s
fieldNameValueSeparator Separator between the field name and value :\\s
lineNumberFormat Format string of line number
