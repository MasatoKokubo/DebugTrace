DebugTrace class has some logging methods.

The following are a sample code used DebugTrace methods and a log of when the sample was executed.
Log bodys are automatically indent.
Sample code:
01 package sample;
02 import jp.masatokoubo.debugtrace.DebugTrace;
03
04 // main
05 public static main(String[] args) {
06   DebugTrace.enter();
07   DebugTrace.println("args", args);
08
09   sub(args);
10
11   DebugTrace.leave();
12 }
13
14 // sub
15 private static sub(String[] args) {
16   DebugTrace.enter();
17
18   DebugTrace.println("args", args);
19
20   DebugTrace.leave();
21 }
Log of when the sample code has been executed:

2016-06-20 19:55:11.001 DBEUG         args:(String[] length:3)["arg0", "arg1", "arg2"] (18)

DebugTrace read DebugTrace.properties file in the classpath on startup. You can specify following propeties in the DebugTrace.properties file.

Properties of DebugTrace.properties file:
NameValueDefault
threadChangeString Format string to output when a thread has been changed
    parameter:
    %1: The thread name
______________________________ %1$s ______________________________
indentString String used for indent.
    "\\s" are change to a space character
\\s\\s\\s\\s
enterString Log string used by enter method
    parameters:
    %1: The class name of the caller
    %2: The method name of the caller
    %3: The line numnber of the call point
Enter %1$s.%2$s(%3$d)
    If your locale is de, es, fr or it, then Enter change to Eingeben, Entrar, Entrer or Entrare.
leaveString Log string used by leave method
    parameters:
    %1: The class name of the caller
    %2: The method name of the caller
    %3: The line numnber of the call point
Leave %1$s.%2$s(%3$d)
    If your locale is de, es, fr or it, then Leave change to Verlassen, Dejar, Quitter or Lasciare.
resultSetRowSeparator Line separator when output a ResultSet object \\r\\n
utilDateFormat Format string when output a java.util.Date object
    parameter:
    %1: a java.util.Date object
 %1$tY-%1$tm-%1$td %1$tH:%1$tM:%1$tS
sqlDateFormat Format string when output a java.sql.Date object
    parameter:
    %1: a java.sql.Date object
 %1$tY-%1$tm-%1$td
timeFormat Format string when output a java.sql.Time object
    parameter:
    %1: a java.sql.Time object
 %1$tH:%1$tM:%1$tS
timestampFormat Format string when output a java.sql.Timestamp object
    parameter:
    %1: a java.sql.Timestamp object
 %1$tY-%1$tm-%1$td %1$tH:%1$tM:%1$tS.%1$tL
arrayMax Max length when output an array 100
byteArrayMax Max length when output a byte array 100
mapMax Max count when output a Map 100
stringMax Max length when output a String 200
resultSetMaxRows Max rows when output a ResultSet object 100
resultSetMaxColumns Max columns when output a ResultSet object 20
logger Logger wrapper class
Select from follows.
    JdkLogger : if you use java.util.logging.Logger
    Log4jLogger : if you use org.apache.log4j.Logger of Log4J
    Log4j2Logger : if you use org.apache.logging.log4j.Logger of Log4J 2
    SystemOutLogger : if you use System.out
